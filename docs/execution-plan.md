# Execution Plan — Rilo Analysis Rebuild

## 0. Mission
在不破坏现有测算能力的前提下，完成 Hopeful Finance → Rilo Analysis 的整体重构：
- 统一信息架构
- 完成视觉重构
- 降低页面/组件/说明系统的耦合度
- 保留三个页面，但形成统一工作台体验

## 1. Workstreams

### WS1. 文档与决策收敛
- 合并多版本 docs 中重叠/冲突内容
- 将用户已拍板决策写入 `docs/decision-log.md`
- 输出一个可执行的单一真相源文档集

### WS2. 结构壳层重构
- 统一页面骨架：参数配置 / 概览 / 敏感度分析
- 统一头部、卡片、控件、结果区、说明入口
- 定义右侧抽屉 / Inspector 的边界

### WS3. 参数配置页重构
- 所有当前可调参数显式可见，不默认折叠
- 重分类、重排版、统一控件
- 页面内即时展示基础结果

### WS4. 概览页重构
- 重组结果层级：收入 / 成本 / 利润 / 关键辅助指标
- 保留有价值的既有 summary 信息，避免无意义删减
- 弱化投资回报率，突出测算结果的可读性

### WS5. 敏感度分析页重构
- 保留“全局敏感度分析”核心定位
- 统一视觉与控件
- 不改其功能本质，避免额外范围膨胀

### WS6. 定义说明系统重构
- 统一中文文案
- 一个轻量按钮触发右侧抽屉
- 不抢主视觉空间
- 与公式中的缩写展示保持一致

### WS7. 财务口径与派生指标校准
- 修正 HQ Fee 基数为总收入
- 校准“每间可售房晚收入”等需展示指标
- 不轻易破坏主计算合同

### WS8. 品牌与目录收尾
- 收尾阶段统一替换 Hopeful Finance 命名
- 视情况调整目录结构与文档结构
- 最终完成品牌层统一

## 2. Cadence

### 2.1 记录规则
在以下时点必须写入工作日志：
- 收到问题
- 开始解决
- 遇到困难
- 做出关键决策
- 解决问题

### 2.2 五分钟节拍
每 5 分钟记录一次：
- 当前子任务
- 当前状态（doing / blocked / done）
- 风险或阻塞
- 下一步动作
- 是否需要快照

### 2.3 快照规则
出现以下情况立即快照：
- 模型明显变慢
- 工具/CLI 不稳定
- 已完成一个较完整子阶段
- 准备进行目录重构或大规模文件移动

## 3. Definition of Done
- docs 收敛到单一主叙事
- 三个页面形成统一工作台体验
- 参数页即时测算反馈成立
- 概览页结果结构清晰、成本信息可见
- 敏感度分析页保留全局定位
- 定义说明通过轻量按钮 + 右侧抽屉触发
- HQ Fee 口径修正为按总收入计提
- 品牌与目录最终完成系统性替换
- `scripts/smoke.sh` / `scripts/review_codex.sh staged` 跑通；`npm test` 如有配置则通过
