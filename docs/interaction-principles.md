# Interaction Principles (Hopeful Finance)

目标：主屏“干净、可直觉操作”，把解释和证明收进 Inspector/tooltip；让高频参数调整是低摩擦的。

## 信息层级（L0 / L1 / L2）

- **L0（主屏默认可见）**：
  - 必填输入、核心结果（年营收/成本/净利/回本）。
  - 简短标签 + 单位；不展示长段解释。
  - 允许“就地修正”（输入框直接改）。

- **L1（轻量解释）**：
  - **hover / tooltip / term**：术语解释、单位口径、简单公式（1 行）。
  - 用来“减少疑惑”，不是用来“指导操作”。

- **L2（证明层 / 过程层）**：
  - **Inspector Panel（右侧）**：公式推导、中间值、口径说明、推理链。
  - 只在用户想核对/质疑时展开，不抢主屏注意力。

## 渐进式披露（Progressive Disclosure）

1. **默认极简**：主屏只显示足够做决策和调整参数的信息。
2. **解释靠近术语，不靠近流程**：用户看不懂某个词 → tooltip/term；用户怀疑结果 → Inspector。
3. **不要为了“解释”增加步骤**：任何解释都不应该阻断输入路径。

## Tooltip / Hover 的边界

- Tooltip 适合：
  - 术语定义（如 COGS、净利润率、回本周期）
  - 口径提醒（例如“LTV 这里先用收入或毛利近似”）
  - 1 行公式（例如“回本=初始投资/年净利润”）

- Tooltip 不适合：
  - 关键操作说明（会被忽略、且移动端不可用）
  - 必须读完才能填的长文本
  - 需要用户做选择/分支决策的内容

> 规则：**tooltip 是辅助，不得取代核心信息与输入。**

## 高频参数的输入策略（尤其 Slider）

### 原则
- **高频参数（Settings 页）必须支持“直接输入数字”**。
- Slider 只能作为“快速探索”的第二通道，不能成为唯一通道。
- 同一个参数不要出现多个互相打架的控件；要么一个双通道控件，要么清晰拆分。

### Slider 选型建议
- 适用：
  - 比例/百分比（0–100%）
  - 范围探索（例如 occ 50–95）
  - 需要快速扫一遍的敏感度参数

- 不适用：
  - 需要精确输入的绝对值（租金/工资/客单）
  - 大范围数值（0–1,000,000）

### 推荐组件形态：Input + Slider 双通道
- 左边/上方：`InputNumber`（可键入）
- 右边/下方：`Slider`（可拖动）
- 两者绑定同一 state：
  - 输入时不会“跳动”
  - slider 的 step 与输入一致

### Range / Percent 统一组件
- 百分比：统一 suffix `%`、统一 step（如 1 或 0.5）
- 区间：统一用 RangeSlider（如需要）并提供两个数字输入

## 默认行为（减少噪音）

- 默认不展示长说明、不展示大量计算过程。
- 右侧 Inspector 保留：
  - 作为“证据层/解释层”
  - 作为“公式词典”（但能 tooltip 的尽量 tooltip 化）

---

最后一句：**主屏是给人做决定的，不是给人上课的。**
