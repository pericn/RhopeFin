# UX / 交互文档 — Rilo Analysis（整体重构）

## 1. 总原则
1) **默认少信息**：先给结论，明细与术语按需展开
2) **单一权威信息源**：术语解释只在一个地方维护，避免 Drawer/面板重复
3) **不减功能，只归位**：所有原功能保留，但重新命名、分区、折叠与层级化
4) **连续反馈**：调参 → 右侧关键结果即时更新

---

## 2. 页面结构（现状 vs 目标）
### 2.1 顶层导航
- Tabs：参数配置 / 财务分析 / 敏感度分析

### 2.2 参数配置页（Settings）— 目标布局
**两栏布局（左输入 / 右解释）**：
- 左栏（Inputs）：
  - 基础设置（Basic）
  - 投资设置（Investment）
  - 成本设置（Cost）
  - 收入设置（Revenue）
  - 自定义模块入口（收入/成本/投资）保持在各自区块内或统一入口（不删）
- 右栏（Inspector Panel）：
  - **结论**：关键结果（年总营收/净利润/净利率/回本）
  - **明细**：公式与明细（收入明细/COGS/成本/回报）
  - **术语**：术语解释（指标字典）

> 默认停留在「结论」（新手友好）；熟手可一键切到「明细」。

### 2.3 财务分析页（Overview）— 目标分区
- 顶部：关键 KPI（大卡片）
- 中部：结构图表（收入结构/成本结构）
- 下部：情景分析预览 + 风险提醒与洞察 +（可选）详细计算过程

> 建议：Overview 不默认展示超长的“详细计算过程”，改为折叠块或“查看计算明细”入口（不删内容）。

### 2.4 敏感度分析页（Analysis）— 目标分区
- 顶部：参数选择/范围/影响指标（控制台）
- 中部：影响图表 + 关键指标卡 + 洞察
- 底部：详细情景对比表

## 3. 信息层级（统一定义）
### 3.1 结论（关键结果）
回答：**算出来多少（结论）**
- 年总营收
- 年净利润
- 净利率
- 回本周期（或 ROI）

规则：
- 默认不展示公式与计算过程
- 允许少量“口径提示”（一句话）

### 3.2 明细（公式与中间变量）
回答：**怎么得出来的**
- 收入明细（会员/寄养/医疗/零售/餐饮/自定义）
- COGS 明细（按业务毛利率推算）
- 固定/变动成本明细
- 投资回报（ROI/回本）

规则：
- 默认不占用新手的第一屏
- 可以按模块折叠（收入/成本/投资/回报）

### 3.3 术语（术语解释）
回答：**这个词是什么意思**
- ADR / Occ / RevPAR / Rooms / Days 等

规则：
- **单一权威内容源**：「术语」Tab 与 Drawer 必须复用同一份内容组件/数据
- “术语解释”入口尽量不弹窗打断：
  - 在 Settings：优先切到「术语」Tab
  - 在 Overview / Analysis：优先切到右侧「术语」；只有在无右侧面板时才 fallback Drawer

---

## 3.4 Hover 规范（术语可解释性）
### 触发点
- Settings / Overview / Analysis 的关键 KPI 与关键输入 label 必须出现可解释术语（浅蓝下划线）
- 至少覆盖：ADR / Occ / Rooms / Days / RevPAR / Profit / Margin / Payback

### 视觉规范
- 术语样式：**浅蓝下划线**（不加 icon，避免噪音）
- 鼠标样式：cursor=help

### 交互规范
- hover 显示 Popover（可交互，不应“一碰就消失”）
- Popover 内容包含：短释义 + 「查看更多」
- 点击「查看更多」：
  - 在 Settings：切换到右侧计算面板的「术语」Tab，并定位/高亮对应条目
  - 在其他页面：优先复用「术语」（若无右侧面板则降级为 Drawer）

### 文案微提示（教程）
- 在 Settings 顶部（标题下方）显示一行提示：
  - 「提示：带浅蓝下划线的术语可悬停查看解释；点『查看更多』可跳到右侧『术语』。」

---

## 4. 命名与文案统一
- “指标字典”统一称为：**术语**
- “计算公式与数据”统一称为：**明细**
- 右侧整体称为：**计算面板**

---

## 5. 默认视图 vs 高级展开
- 默认：结论
- 高级：明细 / 术语
- 触发方式：
  - 右侧 Tabs
  - 顶部“术语”按钮（在 Settings 直接切到「术语」）

---

## 6. 关键交互验收点（UX 级）
- 新手只看「结论」也能完成“调参→判断能否赚钱/回本多久”
- 熟手能在「明细」找到原来所有计算过程，不缺项
- 「术语」内容不重复、不冲突，且入口一致
